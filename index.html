JavaScript

Today, using JavaScript you can create:

  • web applications using frameworks such as React, Vue, Angular and others;
  • back-end applications using Node.js;
  • mobile applications with React Native or Ionic;
  • desktop applications using Electron;

Adding Script

To add a script to a web page, HTML files use the script tag. In its src attribute you should include a link to an external JavaScript file. The script should be added before the <body> closing tag, after all content.

<script src="./js/script.js" type="module"></script>

In order to run your script in strict mode, it is enough to enter the following directive at the beginning of the js-file. Another way to run script in strict mode to use attribute type="module". Always write your code in strict mode.

  • export default 'entity' - to export entity;
  • import varName from './module-name.js' - to import read-only live bindings which are exported by another module.

Syntax Basics

  • Statement is a related set of words and symbols from the syntax of a language that combines to express a common idea, in order to give an instruction to a PC.
  • Expression is a reference to a variable or value, or to a set of variables and values in combination with operators. The statement contains expressions.
  • Interface is a set of properties and methods of an entity available for use in the source code.
    • data has properties, for example, a string has a length. The syntax for accessing a property is very simple: use a dot - entity.propertyName.
    • data has its own predefined methods, this is an action call. The syntax for calling a method is similar, but parentheses are added at the end - entity.methodName().

Variables

Variables are used to store data and consist of an identifier (name) and a memory space for storing their values. A variable declaration begins with the keyword var, let or const:

  • const declaration creates block-scoped constants, the value of it can't be changed through reassignment;
  • let declaration declares a block-scoped local variable, he value of it can be changed through reassignment, latter can only be accessed after its declaration is reached;
  • var statement declares a function-scoped or globally-scoped variable, optionally initializing it to a value, latter can be accessed before its declaration is reached;

Undefined is a variable that has been declared with the keyword let, but has not been initialized with a value.
Undeclared or not defined is a variable that has not been declared in the available scope.

Variable scope is the availability of variables in a certain part of code. Any construct using curly braces {} (conditions, loops, functions, etc.) creates a new local scope, and variables declared in this scope using let or const are not available outside this block.

Primitive types

typeof operator is used to get the variable's value type. It returns in place of its call the value type of the variable specified after it. A variable is not associated with any data type; its value has a type.

  • number - integers and floating-point numbers.
  • string - indexed set of zero or more characters, enclosed in single double or back quotes;
  • boolean - logical data type, true/false flags;
  • null - special value that essentially means nothing, is used in situations where emptiness must be explicitly indicated;
  • undefined - when a variable is declared but not initialized, its value is undefined.

User Input

  • data output methods are console.log() and alert(), the alert() method displays a modal window, with a text corresponding to the value of the variable (or literal) in parentheses;
  • data get from the user methods are prompt() and confirm(), as a result of their execution, they return what was entered by the user, it can be written to a variable for further use (prompt always returns a string).

Logical Operators

In logical operations, the types of operands are converted/cast to true or false. Truthy and Falsy are the terms used for those values that are converted to true or false in a logical operation, although they were not originally booleans. Values that are converted to false in logical conversion 0, NaN, null, undefined, empty string, false.

  • Boolean() function - to convert a value to logical true or false;
  • Logical AND && - result will be false, since at least one of the operands is converted to false, in opposite returns last true value;
  • Logical OR || - result will be true, since at least one of the operands is converted to true, in opposite returns last false value;
  • Logical NOT ! - replaces value with the opposite.
  • Nullish coalescing operator ?? - returns its right-hand side operand when its left-hand side operand is null or undefined, and otherwise returns its left-hand side operand;
  • Logical OR assignment ||= - (x ||= y) operator only assigns if x is falsy.

Branches and Cycles (Lesson 2)

Branching

Branches are used to execute particular code depending on the condition. The principle is simple: the condition result is converted to a boolean, true or false, after which the program executes this or that branch.

  • if (condition) {//if body} statement - if the condition is converted to true, the code in curly braces of the if body is then executed;
  • if (condition) {//if body} else {//else body} statement - if syntax is extended, if the condition is converted to false, the code in curly braces after the else statement will be executed;
  • else if statement - enables you to add after else another if statement with a condition.

Ternary Operator

condition ? expression_if_true : expression_if_false - the ternary operator is used only for return and assign operations as a more syntactically concise replacement for the if ... else statement.

Switch Statement

The scope of switch is limited to tasks with one common question (what to compare) and many answer options (what to compare with). For comparison, the strict equality operator === is used.

switch (value) {
case value: statements;
break;
case value: statements;
break;
default:
statements;
}

Functions (Lesson 4)

Function declaration, expression, calling

Function is a subroutine, an independent piece of code designed to repeatedly execute a specific task with different initial values. Function name must be a verb answering the question "What to do?". If you need to declare a function with parameters other than undefined, it is enough to specify the default value.

  • Function declaration function functionName(parameters) {\\function body} - can be called before its creation;
  • Function expression const functionName = function(parameters) {\\function body} - cannot be called before its creation, only after, as it is literally a const variable declaration;
  • Function calling functionName(arguments) - in parentheses you can pass arguments, i.e. values for the declared function parameters.

Usage Features

  • arguments can be accessed using the special variable, which is only available inside the function and stores all arguments as a pseudo-array:
    function multiply() {let total = 1; for (const varName of arguments) total *= varName; return total;}

    function fnName(...argToArray) {There is access to Array 'argToArray' that was made from arguments}
  • return operator passes a value from the function body to external code and exit the function:
    function multiply(x, y, z) {return x * y * z;};
  • return early pattern is a way to return early from a function using the return operator:
    function withdraw(amount, balance) {
    if(amount === 0) {
    console.log("Enter an amount greater than zero");
    return;}
    if (amount > balance) {
    console.log("Insufficient account balance");
    return;}
    console.log("Withdrawal has been successful");}

Scope

Scope is a mechanism that determines the availability of variables in your code. Scope chain: scopes form a hierarchy so that child scopes can access variables from parent scopes, but not vice versa.

  • Variables declared at the very top level, that is, outside of any constructs like if, while, for and functions, are in the global scope, and they are available everywhere after they are declared.
  • Variables declared inside if and for statements, functions and other blocks of code enclosed in curly braces {} are in the block scope, and they are available only within this block of code or those nested inside it.
  • Scope chain look-up - the interpreter first tries to find a variable in the scope in which such variable was accessed. If there is no such variable in the local scope, then the interpreter goes out, one level per attempt, until it finds the necessary value or reaches the topmost scope (global) and realizes that no variable with such an identifier can be found, since it does not exist

Call Stack

Call stack is a mechanism for keeping track of where the interpreter is located in the code that calls multiple functions. You can see which of the functions is currently being executed, which functions are called from within the function being executed, which will be called next, etc. Stack is a data structure using the LIFO (Last-In-First-Out) principle. The last thing that is added to the stack will be removed from it first, which means that you can add or remove elements only from the top of the stack.

Objects (Lesson 5)

Object properties and methods

Objects serve to describe and group the characteristics of an entity. Objects are also called dictionaries, that is, they contain terms (properties) and their definitions (values). Objects can store not only data, but also functions for working with this data, i.e. methods. If the property value is a function, such property is called an object method. Such objects can be called models. They couple data with methods to work with that data.

  • const object = {propertyKey: value, ...} - object creation, property value can be another object in order to store nested and grouped data;
  • const object = {variableName, ...} - shorthand object creation, when creating an object and you need to take the property value from a variable or function parameter of the same name as the property;
  • const object = new constructor(arguments) - creates an instance of a user-defined object type or of one of the built-in object types that has a constructor function, constructor data property of an object returns a reference to the constructor function that created the instance object;
  • Object.create(proto[, propertiesObject]) method - creates a new object, using an existing object as the prototype of the newly created object;
  • Object.assign(target, source1, source2, ...) method - copies all enumerable own properties from one or more source objects to a target object;
  • object.propertyKey - to access a value of the proherty;
  • object.propertyKey.nestedPropertyKey - to access a value nested the property;
  • object.propertyKey = value - adding objects property whith value or changing objects property value;
  • delete object.propertyKey or delete object[propertyKey] - removes a property from an object;
  • object["propertyKey"] - to access a value of the proherty;
  • object[variableName] - to access a value of the property with a name that you do not know, as it is stored as the value of a variable;
  • [variableName] - to access a property with a name that you do not know, as it is stored as the value of a variable;
  • const object = { [variableName]: value, ...} - to add objects property with a name that you do not know, as it is stored as the value of a variable or as the result of a function, the value of a computed property can be any valid expression;
  • fnName: function() {\\Body} or fnName() {\\Body} - to store function (object method) for working with the object's (model's) data;
  • this.propertyKey - to access the properties in object methods, we refer to it through this and then, as usual, to its properties using dot notation.

Destructuring and spread/rest (Lesson 6)

Spread and Rest Syntax

  • ... (spread) passing arguments - to spread a collection of elements from array, string or object to the places where a set of single values is expected;
  • ... (spread) creating a new array - to create a copy of an array or "bond" an arbitrary number of arrays together to make a new one:
    const copyOfArray = [...array];
  • ... (spread) creating a new object - to spread the properties of an arbitrary number of objects to a new one:
    const third = {propertyName: value, ...first, ...second }, it is possible add properties to any location, spread order matters, object property names are unique, so the properties of the sprayed object can overwrite the value of an existing property if their names are the same;
  • ... (rest) collecting all function arguments - to collect a group of independent elements into a new array;
  • ... (rest) collecting part of function arguments - to collect into an array only the required part of the arguments:
    function fnName(firstNumber, secondNumber, ...otherArgs);

Object Destructuring

Destructuring enables you to "unpack" the values of an object's properties into local variables, if the variable and property names are the same, assignment takes place, otherwise it will be assigned undefined.

  • const {varName1, varName2, varName3 = value} = objectName - specifies default values,they will be assigned only if the object does not have a property with that name;
  • const {propertyName1: newPropertyName1, propertyName2: newPropertyName2} = objectName - changing variable name;
  • for (const book of books) {const {title, author, rating} = book;} - destructuring in loop body;
  • for (const {title, author, rating} of books) {} - destructuring in loop can be done right where the variable book is declared;
  • const {propertyName1, propertyName2: {nestedPropName1, nestedPropName2}, propertyName3} = objectName - deep destructuring is for the properties of nested objects.;

Array Destructuring

Destructuring assignment can be used for arrays as well: use square brackets [] instead of curly braces {}.

  • const [varName1, varName2, varName3] = arrayName - creates variables with elements placed in them in ascending order, from 0 to the array end;
  • let varName1, varName2, varName3; [varName1, varName2, varName3] = arrayName - value can be assigned to a variable after arrays declaration;
  • const [varName1, varName2, varName3 = value] = arrayName - specifies default values;
  • const [varName, ...restArrayName] = arrayName - unpacks and assigns the rest of the array elements to a variable;
  • const [, , varName] = arrayName - skips elements.

«Parameter Object» Pattern

If a function has more than two or three arguments, it is very easy to get confused about what is passed and in which order. The parameter object pattern solves this problem by replacing a set of parameters with just one, an object with named properties:

  • function fnName(objectName) {\\Function body} - then, when calling, you pass one object with the necessary properties;
  • function fnName(objectName) {const {varName1, varName2} = objectName; \\Function body} - destructure the object in the objectName parameter;
  • function fnName({varName1, varName2}) {\\Function body} - destructure the object in the function signature.

Callbacks and Arrow Functions (Lesson 7)

Callbacks

Callbacks are used to process user actions on the page, process requests to the server, execute unknown functions, etc. and intended for deferred execution.Any function can assume an arbitrary number of callbacks.

  • Callback is a function that is passed as an argument to another function, and the latter calls such function:
    function callbackName(element) {\\Logic whith element};
    higherOrderName(element, callbackName);
  • Higher-order function is a function that takes other functions as parameters or returns a function as a result:
    function higherOrderName(element, callback) {\\Logic whith element; callback(element);}
    higherOrderName(element, callbackName);
  • Inline callback is a function declared immediately when calling the function to which the callback is being passed and will be available only as a parameter value and nowhere else in the code:
    higherOrderName(element, function callbackName(element) {\\Logic whith element};);

Arrow Functions

Arrow functions are created as a function expression, and if the function is not anonymous then it must be assigned to a variable.

  • const arrowFn = (param1, param2, param3) => {\\Fn body} - several parameters;
  • const arrowFn = param1 => {\\Fn body} - one parameter;
  • const arrowFn = () => {\\Fn body} - no parameters;
  • const arrowFn = (a, b, c) => {return a + b + c;} - explicit return when there are curly braces, and the function must return a value;
  • const arrowFn = (a, b, c) => a + b + c - implicit return will return the result of expression;
  • const arrowFn = (...args) => {\\Fn body} - there is no local variable arguments inside, to collect all arguments into an array, the rest operation is used.

Code Types

  • Imperative programming describes computation as a set sequence of statements that change a program state. It is a description of how something is done.
  • Declarative programming describes what you want as a result, not how to achieve it. The order of execution and the method of achievement are not important.

Array Iteration Methods (Lesson 8)

JavaScript has array methods that come from functional languages. Most of them are pure functions. They create a new array, populate it, applying the specified callback to the value of each element, and then return this new array. Side-effect function is a function that, during execution, can change or use global variables, change the values of reference arguments, perform I/O operations, etc. Pure function is a function with the result depending only on the values of the passed arguments and has no side effects, that is, it does not change the values of the arguments.

Syntax

  • array.method(callbackFn[, thisArg]) - callback function;
  • array.method(function callbackFn(element, index, array) {//logic for each iteration}[, thisArg]) - inline callback function;
  • array.method((element, index, array) => {//logic for each iteration}) - arrow function.

  • .flatMap() method is similar to the map() method, but it is used in cases where the result is a multidimensional array that needs to be "flattened", tnew array is "flattened" by one (one nesting);
  • .filter() method - to copy array whith filtering, when more than one element from a collection should be selected using a specific criterion;
  • .find() method - to find and return the first matching element, after which the array iteration stops;
  • .findLast() method - to find and return the last matching element, after which the array iteration stops;
  • .findIndex() method - to find and return the index of the first element that satisfies a condition, that is, when the callback returns true;
  • .every() method - returns true if all array elements satisfy a condition, otherwise returns false;

The keyword this (Lesson 9)

Function Execution Context

The reserved keyword this can be used inside functions. During the execution of a function, a reference to the context object of its call is written to this. Therefore, in the function body, you can access the properties and methods of this object. The context value inside a function (not arrow) is determined not at the time of its creation, but at the time of calling. That is, the value of this is determined by how the function is called, not where it was declared.

  • In the global scope, in non-strict mode, this refers to the window object, in strict mode the value of this in the global scope will be undefined.
  • In object method, the context (this) will refer to the object that includes the method.
  • In callback functions the context (this) is not preserved.
  • Arrow functions do not have their own this. The context inside an arrow is determined by the place of its declaration, not the call, and refers to the parent function’s context. Arrow functions also ignore the presence of strict mode. If the arrow remembers the global context, this in it will contain a reference to window, regardless of whether the script is executed in strict mode or not.

Function Methods

There are situations where a function needs to be called in the context of an object, and the function is not its method. To do this, functions have call, apply and bind methods.

  • .call(objectName[, arg1, arg2, ...]) method calls the function that contain a reference to the objectName object, and passes arguments as arg1, arg2;
  • .apply(objectName[, [arg1, arg2, ...]]) method calls the function that contain a reference to the objectName object, and passes an array as arguments;
  • .bind(objectName[, arg1, arg2, ...]) method creates and returns a copy of the function with the bound objectName context and such arguments as arg1, arg2, the function can be passed anywhere and called whenever.

Object-Oriented Programming

  • Procedural programming is a set of not explicitly related functions and variables for storing and processing information.

    • Instance (object) is a separate representative of a class with a specific state and behavior completely defined by the class.

DOM (Lesson 11)

Document Object Model

  • Document Object Model is a language-independent interface for work with HTML documents. It offers a set of properties and methods to search for, create and delete elements, respond to user actions, and much more. In other words, it serves as a link between the page and the programming language.
  • Browser Object Model is a language-independent interface for work with browser tabs. It offers a set of properties and methods to access directly the current tab and a number of browser functions. This model includes an object to work with navigation history, location, and more./li>

According to the DOM model, each tag forms a separate node element, whereas each piece of text is a text element. An HTML document is a hierarchical tree in which every element (except for the root) has only one parent. Browser engines have a special piece of code called an HTML parser that is used to convert HTML to DOM. After the DOM tree is ready, you can find elements using JavaScript and perform some actions with them, since each element has an interface with many properties and methods.

Events (Lesson 12)

Events

An event is a signal from the browser that something has happened on a web page. In order for an element to react to user actions, it needs an event listener (handler), that is, a function that will be called as soon as the event occurs.

  • elem.addEventListener(event, handler[, {\\options}]) - dds an event listener to the elem element, where:
    • event is event name, a string, for example 'click';
    • handler is a callback that will be called when the event occurs;
    • options is an optional parameter object with advanced options: {once: true} - removes the event listener after its execution.
  • elem.removeEventListener(event, handler[, {\\options}]) - removes an event listener from the elem element, it is need to pass a reference to the callback that was assigned in addEventListener();
  • elem.addEventListener(event, objName.callbackName.bind(objName)) - if the callback is a function that uses this, by default the context inside it will refer to the elem element with the listener.

Event Object

Each event is an object that contains information about the event details, and it is automatically passed as the first argument to the event handler. All events belong to the base class Event with properties and methods:

  • event.type property is event type;
  • event.target property is the element on which the event is occured;
  • event.currentTarget property is the element on which the event handler is executed;
  • event.preventDefault() method is to cancel the default browser action on the event object;
  • event.stopPropagation() method impedes the event to go further;
  • event.stopImmediatePropagation() method is to completely stop event handling.

KeyboardEvent Object

Unlike others, keyboard events are handled on the document, not on a specific element. There are three basic keyboard events: keydown, keypress and keyup. KeyboardEvent objects belong to the KeyboardEvent base class:

  • event.key property returns the character generated by key pressing, adjusted for the state of modifier keys, such as Shift, and the active language;
  • event.code property returns the code of a physical key on the keyboard and does not depend on languages.
  • event.ctrlKey(.altKey | .shiftkey | .metaKey) property storing a boolean value indicating whether a modifier key is pressed or not.

Form Events

  • 'click' event occurs when a pointing device button (such as a mouse's primary mouse button) is both pressed and released while the pointer is located inside the element;
  • 'submit' event can be used to validate (check) a form before submitting, when you click on a button with the type ="submit" attribute or press the Enter in any of its text fields;
  • 'change' event occurs after a form element has changed, useful for checkboxes and radiobuttons;
  • 'input' event occurs only in text fields and textarea, and it is generated every time the element value changes, without waiting for focus to be lost;
  • 'focus' and 'blur' events occur when an element is focused and when focus is lost, these two events do not bubble;
  • 'focusin' and 'focusout' events occur when an element is focused and when focus is lost, the two events differ in that they bubble;

Event Delegation (Lesson 13)

Event Propagation and Delegation

  • Event propagation is a term describing the life cycle of an event, which includes three stages: capturing, targeting and bubbling. If there is an event listener registered on the outermost element, it will "catch" all clicks inside, because events will bubble up to that element. Typically, the event will bubble up to the window element, calling all handlers on its way. Do not stop bubbling unless necessary.
  • Bubbling makes it possible to do one of the most useful tricks, event delegation. When there is a group of elements with events that need to be handled in the same way, then one handler should be added to their common ancestor, instead of adding a handler to each element.
